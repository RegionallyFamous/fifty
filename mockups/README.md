# `mockups/` — concept queue source

Every file in this folder is a *concept on the bench* — a mockup of a
storefront the build agents have not yet shipped. The static GitHub
Pages site at <https://demo.regionallyfamous.com/concepts/> renders
this folder as the public queue.

## Layout

Two layouts are supported. Pick the simpler one until you need the
richer one.

```
mockups/
├── README.md               (this file)
├── mockup-<slug>.png       (single-image concept)
├── <slug>.meta.json        (paired metadata; required for every concept)
└── <slug>/                 (multi-image concept, OPTIONAL)
    ├── home.png            (required if directory form is used; treated as hero)
    ├── pdp.png             (optional; product detail page)
    ├── cart.png            (optional)
    └── mobile.png          (optional)
```

* The single-image form (`mockup-<slug>.png`) and the directory form
  (`<slug>/home.png`) are mutually exclusive — pick one per concept.
  Both forms coexist across the folder.
* `bin/build-redirects.py` discovers concepts by walking this folder
  and pairs each one with `<slug>.meta.json`.

## `<slug>.meta.json` schema

Every concept **must** ship a sibling `.meta.json`. The file is
generated by `bin/build-concept-meta.py` from the source-of-truth
table at `bin/concept_seed.py` plus the auto-extracted palette from
`bin/extract-palette.py`. Hand-editing the JSON is fine for
one-off tweaks, but the next `build-concept-meta.py` run will overwrite
your changes — push them upstream into `concept_seed.py` to make them
stick.

```json
{
  "slug": "cobbler",
  "name": "Cobbler",
  "blurb": "Bench-made footwear, Goodyear-welted: oxblood + cream, leather + brass eyelets.",
  "tags": {
    "palette": ["oxblood", "cream", "brown", "brass"],
    "type": "oldstyle-serif",
    "era": "pre-1950",
    "sector": "footwear",
    "hero": "photo-hero"
  },
  "palette_hex": ["#f4ecd8", "#5a1f1a", "#1a1a1a", "#7c5b34", "#cba56b"],
  "type_specimen": "Display: ITC Caslon. Body: Crimson Pro."
}
```

### Field reference

| field           | type    | notes                                              |
| --------------- | ------- | -------------------------------------------------- |
| `slug`          | string  | Matches the file name; lower-kebab.                |
| `name`          | string  | Display title shown on cards + detail page.        |
| `blurb`         | string  | One sentence. Lead paragraph + thumbnail alt text. |
| `tags.palette`  | array   | 2–4 color-family tokens (controlled vocab).        |
| `tags.type`     | string  | One type-genre token (controlled vocab).           |
| `tags.era`      | string  | One era token (controlled vocab).                  |
| `tags.sector`   | string  | One sector token (controlled vocab).               |
| `tags.hero`     | string  | One hero-composition token (controlled vocab).     |
| `palette_hex`   | array   | 5 dominant hex colors auto-extracted from the PNG. |
| `type_specimen` | string  | "Display: Foo. Body: Bar." build-agent hint.       |

The controlled vocabularies live as Python `set[str]` constants in
[`bin/concept_seed.py`](../bin/concept_seed.py). Adding a new token is
fine — pick a tight, orthogonal name; rerun the audit + meta-backfill;
commit `concept_seed.py` and the regenerated artefacts together.

## Workflow when adding a concept

1. Drop `mockup-<slug>.png` into `mockups/` (or create `mockups/<slug>/home.png`
   if you have multiple views).
2. Add an entry to `CONCEPTS` in `bin/concept_seed.py`. Keep blurbs to
   one declarative sentence.
3. Run `python3 bin/build-concept-meta.py` to write/update the
   `<slug>.meta.json` (palette_hex is auto-extracted).
4. Run `python3 bin/audit-concepts.py` to refresh `docs/concepts/AUDIT.md`.
5. Run `python3 bin/build-redirects.py` to regenerate the concept queue
   page and per-concept detail pages.
6. Commit everything in one go.

## Workflow when removing or replacing a concept

1. Delete the PNG (or the slug directory) and the `.meta.json`.
2. Delete the matching `CONCEPTS` entry in `bin/concept_seed.py`.
3. Re-run `bin/build-concept-meta.py`, `bin/audit-concepts.py`,
   `bin/build-redirects.py`.
4. The `docs/mockups/<slug>.png` copy and the `docs/concepts/<slug>/`
   detail page will both vanish from `docs/` automatically — that
   directory is fully regenerated on every `build-redirects.py` run.
